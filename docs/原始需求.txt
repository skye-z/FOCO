第二部分：核心功能实现（必须完成）
请完成以下 2 个基础功能。
必做功能 1：考试知识体系与题库管理系统
需求：
支持基础角色设计：管理员、教研/内容编辑、学习者
管理员可管理考试结构与用户权限；教研/内容编辑可维护题目与解析；学习者只能查看已发布内容并参
与练习
支持考试、科目、章节、知识点、题目的层级管理
每道题需包含题型、难度、答案、解析，并关联一个或多个知识点
支持题目批量导入，以及按考试、科目、章节、知识点查询题目
技术要求：
使用 Supabase（PostgreSQL）建模
提供题库管理 API 与最小可用前端管理界面
设计合理索引，支持高频题库查询
必做功能 2：练习、判题与学习记录系统
需求：
学习者可以选择考试、科目、章节或知识点开始练习
支持单选题、多选题、判断题中的至少 2 种题型
提交答案后返回判题结果、正确答案、解析、关联知识点和答题耗时
自动记录答题行为，错题进入错题本
支持查看基础进度，例如已完成题数、正确率、最近学习日期、错题数量
技术要求：
设计答题 session 与答题记录数据结构
前端需要有完整练习流程
后端需要保证重复提交、刷新页面、网络重试时的数据一致性
提供答题结果、学习记录、错题查询 API
需要为答题记录写入、错题归档逻辑编写单元测试
第三部分：学习智能化进阶（三选二）
从以下 3 个进阶方向中选择 2 个进行实现。
进阶方向 1：每日学习路径与习惯留存系统
目标：把长期备考拆成短小、明确、每天可完成的学习路径。
需求：
学习者每天进入后看到一条明确的今日学习路径
今日路径包含 2-4 个短任务，例如新题、错题、复习、概念回看
系统根据最近表现调整今日路径，而不是固定任务清单
每完成一步都有即时反馈，例如进度推进、XP、完成状态或奖励
支持连续学习记录、中断保护或恢复任务
展示最近 7 天学习节奏
技术要求：
设计学习路径、任务步骤、学习事件、连续学习记录的数据结构
路径生成需考虑考试日期、最近表现、错题和未完成任务
任务完成、XP/奖励、连续学习计算需要幂等
前端需展示今日路径、逐步完成反馈和连续学习状态
进阶方向 2：诊断测评、知识图谱与自动任务选择引擎
目标：用诊断和知识点图谱判断学习者当前水平，由系统选择下一步最值得学习的任务。
需求：
支持入门诊断测评，生成初始强弱项
建立知识点图谱，包含知识点、前置关系和题目映射
根据诊断和答题记录维护学习者知识画像
自动选择下一步任务：新学、补前置、错题复习或间隔复习
每条任务必须说明选择依据，例如前置未掌握、近期答错、接近遗忘周期
技术要求：
设计诊断测评、知识图谱、知识画像、任务选择记录的数据结构
任务选择逻辑需要可解释，并有单元测试
说明如何在新学、补前置、复习、推进章节之间取舍
前端需展示知识点状态、推荐任务和推荐原因
进阶方向 3：交互式概念探索与即时反馈系统
目标：把复杂概念或题目做成可交互学习单元，让学习者通过操作、判断和反馈建立理解。
需求：
至少实现 1 个交互式学习单元，围绕一个复杂知识点或题目展开
学习单元需包含 3-6 个交互步骤，例如拖拽公式组件、调整图表/计算参数、排序推理链、匹配概念卡
片、标记关键条件、选择或填空
每一步根据用户操作即时反馈，而不是最后统一给解析
答错时提供当前步骤提示，并解释可能误区
最后生成概念卡片或总结
技术要求：
可以使用 OpenAI API 分析学习数据与题目内容
结构化保存交互步骤、用户操作、反馈、提示和概念卡片
如使用 AI，输出需包含步骤、交互类型、反馈、误区解释等字段
如使用 AI，输出需要结构化保存，并保留生成依据和版本
如使用 AI，需要处理调用失败、超时、返回格式异常等情况
前端必须实现真实交互流程，不能只展示分段文字解析
